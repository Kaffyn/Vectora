KEY RESPONSIBILITIES BY LAYER
==============================

Backend (Go):
- cmd/vectora: Main daemon orchestrator
- internal/infra: Config, logging, environment
- internal/ipc: Client-daemon communication
- internal/core: Business logic, RAG pipeline
- internal/db: Data persistence (bbolt, chromem-go)
- internal/llm: LLM provider integration
- internal/acp: Tool registry and execution
- pkg/*: Shared types and utilities

Frontend (Next.js):
- internal/app/*: React components, pages, state
- TypeScript for type safety
- Tailwind CSS for styling
- Custom hooks for business logic

Desktop Integration:
- cmd/vectora-app: Wails wrapper for desktop app
- cmd/vectora-installer: Windows wizard installer
- IPC for daemon communication

TECHNOLOGY STACK
================

Backend:
  - Go 1.22+
  - bbolt (embedded key-value store)
  - chromem-go (vector database)
  - Gemini & Qwen LLM APIs

Frontend:
  - Next.js 15+
  - React 19+
  - TypeScript
  - Tailwind CSS

Desktop:
  - Wails (Go + Web Framework)
  - Fyne (Installer GUI)

Communication:
  - JSON-ND over Unix sockets/Named pipes
  - gRPC for services

DEVELOPMENT WORKFLOW
====================

1. Backend Development:
   - Implement features in internal/
   - Update pkg/types for new types
   - Write tests in tests/
   - Build with: make build or build.ps1

2. Frontend Development:
   - Develop in internal/app/
   - Use Next.js dev server: npm run dev
   - TypeScript strict mode enabled

3. Integration:
   - IPC protocol in internal/ipc/
   - Service definitions in pkg/types/

4. Testing:
   - Unit tests alongside code
   - Integration tests in tests/integration/
   - E2E tests in tests/e2e/

5. Deployment:
   - Windows installer via vectora-installer
   - Cross-platform builds in build/
   - Auto-update mechanism to be implemented
